# 🎯 프롬프트 사용 가이드

## 📌 프롬프트는 이제 제한이 없습니다!

AI-ERP의 프롬프트는 **상황과 의도에 따라 자동으로 동작**합니다.

---

## 🎨 프롬프트 동작 방식

### 1️⃣ 메뉴가 없을 때
**→ 항상 새 화면 생성**

```
✨ 무엇을 만들까요? (예: 사용자 등록 화면 만들어줘)
```

**예시:**
- "사용자 등록 화면 만들어줘"
- "제품 재고 관리 화면 생성해줘"
- "주문 관리 시스템 만들기"

---

### 2️⃣ 메뉴를 선택한 후

**→ 의도에 따라 자동 판단!**

```
💬 데이터 입력 (예: 홍길동, hong@company.com) 
   또는 새 화면 생성 (예: 제품 관리 화면 만들어줘)
```

#### A) 데이터 추가 의도
**키워드 없이 데이터만 입력**

```
홍길동, hong@company.com, 010-1234-5678
→ 현재 화면에 데이터 추가 ✅
```

#### B) 새 화면 생성 의도
**"화면", "만들어", "생성" 등 키워드 포함**

```
제품 관리 화면 만들어줘
→ 새 화면 생성 ✅
```

---

## 🔑 키워드 감지 시스템

### 새 화면 생성 키워드
다음 키워드가 포함되면 **새 화면을 생성**합니다:

- **한글**: 화면, 만들어, 생성, 추가해줘, 만들기
- **영문**: screen, create

### 예시

#### ✅ 새 화면 생성
```
"제품 관리 화면 만들어줘"          → 새 화면
"주문 시스템 생성해줘"             → 새 화면
"고객 관리 추가해줘"               → 새 화면
"Create user screen"             → 새 화면
```

#### ✅ 데이터 추가
```
"홍길동, hong@company.com"        → 데이터 추가
"노트북, 1500000, 50개"           → 데이터 추가
"주문번호 ORD-001, 홍길동"        → 데이터 추가
"Add user hong"                  → 데이터 추가
```

## 💡 실전 사용 예시

### 시나리오 1: 사용자 관리 시스템

```bash
# 1. 첫 화면 생성
프롬프트: "사용자 등록 화면 만들어줘"
→ 사용자 등록 화면 생성 ✅

# 2. 데이터 추가
프롬프트: "홍길동, hong@company.com, 010-1234-5678"
→ 사용자 데이터 추가 ✅

# 3. 더 많은 데이터
프롬프트: "김철수, kim@company.com, 010-9876-5432"
→ 사용자 데이터 추가 ✅

# 4. 다른 화면도 만들기
프롬프트: "제품 관리 화면 만들어줘"
→ 제품 관리 화면 생성 ✅ (사용자 화면은 그대로 유지)

# 5. 제품 데이터 추가
프롬프트: "노트북, 1500000, 50개"
→ 제품 데이터 추가 ✅
```

---

### 시나리오 2: 여러 화면 동시 관리

```bash
# 사용자 화면 선택 상태에서
프롬프트: "홍길동, hong@company.com"
→ 사용자 데이터 추가 ✅

# 같은 상태에서 새 화면 만들기
프롬프트: "주문 관리 화면 만들어줘"
→ 주문 관리 화면 생성 ✅

# 주문 화면으로 자동 전환됨
프롬프트: "ORD-001, 홍길동, 노트북, 2대"
→ 주문 데이터 추가 ✅

# 다시 사용자 화면으로 전환 (사이드바 클릭)
프롬프트: "이영희, lee@company.com"
→ 사용자 데이터 추가 ✅
```

## 🎭 특수 케이스

### 1. 애매한 경우
```
프롬프트: "홍길동 화면"
→ "화면" 키워드 포함 → 새 화면 생성 시도
```

**해결책**: 명확하게 표현
```
데이터 추가: "홍길동 추가"
새 화면: "홍길동 관리 화면 만들어줘"
```

### 2. 자연어 입력
```
프롬프트: "홍길동 부장님 이메일은 hong@company.com이고 
          전화번호는 010-1234-5678입니다."
→ "화면" 키워드 없음 → 데이터 추가 ✅
```

### 3. 여러 줄 데이터
```
프롬프트: 
홍길동, hong@company.com, 010-1234-5678
김철수, kim@company.com, 010-9876-5432
이영희, lee@company.com, 010-1111-2222
→ 각 줄을 개별 데이터로 추가 ✅
```

## 🚀 고급 팁

### 1. 빠른 화면 전환
```
사이드바에서 메뉴 클릭 → 즉시 해당 화면으로 전환
프롬프트는 자동으로 해당 화면의 데이터 입력 모드
```

### 2. 메뉴 없이 시작
```
사이드바 빈 공간 클릭 → 메뉴 선택 해제
프롬프트는 새 화면 생성 모드로 전환
```

### 3. 키워드 활용
```
확실하게 새 화면: "○○ 화면 만들어줘"
확실하게 데이터: 키워드 없이 데이터만 입력
```

### 4. 파일 업로드
```
엑셀 파일 드래그 앤 드롭
→ 자동으로 현재 화면에 데이터 추가
→ 메뉴가 없으면 업로드 불가 (먼저 화면 생성 필요)
```

---

## 📋 요약

### 동작 규칙
1. **메뉴 없음** → 항상 새 화면 생성
2. **메뉴 있음 + 키워드 있음** → 새 화면 생성
3. **메뉴 있음 + 키워드 없음** → 데이터 추가

### 키워드 목록
```
화면, 만들어, 생성, 추가해줘, 만들기, screen, create
```

### 우선순위
```
키워드 감지 > 메뉴 선택 상태
```

```
# 명확한 의도 표현

"홍길동 부장님 정보 추가"    (데이터)
"제품 관리 시스템 생성"      (새 화면)
```

### ⚠️ 주의
```
# 애매한 표현
"홍길동 화면"  → 새 화면으로 인식될 수 있음
"데이터 추가"  → 실제 데이터가 없어서 실패

# 해결: 명확하게
"홍길동 데이터: hong@company.com, 010-1234-5678"
"홍길동 관리 화면 만들어줘"
```
